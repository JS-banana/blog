2021年终总结（技术篇）

# 起笔

晃晃悠悠，一年又一年，光阴荏苒，日月如梭

这一年给我的感觉就是“快”，尤其的快，现在提笔写年终，但停下细想，仿佛2021新年目标才刚刚写完，而现在就已经来到了2022年...

我依稀感觉的到自从大学毕业，迈入社会、开始工作后，仿佛一切都开始加速起来了。 好似今天才是周一明天就已经周五了，你说周末？那就更快了，眼睛一眨，咔的一下，周一了...

有人说，充实的日子过的很快，有人说，无聊恍惚的日子过的很快。我希望我的这一年是没有浪费的，是产生了些许价值的，并有所收获，有意义的，至少对于我个人来说须是没有虚度的！

2021年的年终总结，目前看来动笔算是晚了些许，今天已经一月5号了，我才开始动笔（苦笑~），即便如此，我也不觉的今天就可以把我这一年一口气写完...咳~，再多啰嗦几天吧

# 回望

# 从2020

记得是2020年10月份左右，趁活动搞了台服务器。

初衷，是想着学习下服务器相关知识（Linux、Nginx、Docker、NodeJs服务等），作为前端开发工程师,咱也不能只单纯的停留在浏览器上玩呀，一些后端知识还是有必要了解下的。心里也是一直想搞个自己的博客网站玩，正好也是个机会。说干就干，直接开整

然后就慢慢开始搭建搞起了自己的个人博客，那时候也是一腔热血，充满兴趣。经常是下班回家后，就在那鼓弄，可以很投入的忙到很晚（其实就是比较菜😂），毕竟遇到问题都是网上搜，走了不少弯路（如果有前辈带带可能好很多吧），不过这个阶段也完全是兴趣和需求驱使，还挺有行动力的，一直在想着怎么优化性能、解决问题~

总的来说去实战，去写项目，是比较有效果的，遇到问题，解决问题，边做边学，期间也可以做个记录和笔记啥的，印象会比较深刻些，如果只是单纯的看文章和视频，而不去自己尝试的话，效果可能并没有那么好，一定要实操下🤪

还依稀有点印象，那时候对着2021年的新年规划啥的，现在已经记不清了...

# 入2021

对于2021年的规划

当时的想法是，出于自身情况和现实考虑，就目前而言，冲击大厂还是有不少距离的，曲线方式就是继续锤炼自己，提升自身能力，且很有必要向一些大佬和前辈请教学习，因此，这也是接下来主要方向:

  • 一个技术氛围浓厚的团队
  • 可以参与交流学习的社群
  • 持续学习和积累

因此也在年前换了份工作，来到了一个新的环境，这里有着一定规模的技术团队和牛逼的前辈带路，有幸和他们一起工作并从中学习，也少走了一些弯路，这一年过来也是有所成长吧（感觉有大佬带还是很有必要的，对于个人的技术成长也很关键）。要说还没达到自己期望是有些贪心了，重要的也是找到自己的节奏，什么事情也不是一蹴而就的，目标既定，就要按照这个方向坚持，持续积累🧐！

工作中所带来的挑战也是只增不减，会接触到不少复杂的需求，这就需要花费更多的时间去解决。当你不能再依靠现有的能力解决问题时，也就是学习和进步的时候了。这一年下来收获是有一些的，但留给个人的时间确实有所压缩，比较遗憾的就是健身计划几乎一直中断，裂开😶~

整个上半年在工作上算是比较忙的，更多的是在实际业务需求中历练成长了，同时沉淀着技术，当然，也会忙里偷闲写点东西啥的。但是，就目前而言，自己对于技术打磨和学习提升感觉还是有所不足的，这也是上半年比较可惜的点，希望明年可以做的更好（甘巴爹🤓）！

进入下半年之后，时间稍算充裕了一些，有了一些学习和提升自己的时间，也是慢慢渐入状态吧，看了一些文章和博客，也跟着大佬一起简单的读了些源码，获益匪浅。对于以前一知半解的东西也更加深刻了，也有尝试自己写一点东西加深印象。就目前看来，这些方面都是可以继续加强和深入的方向。

源码方面，推荐若川大佬-lxchuan12的读源码系列，能学到很多东西，这一年也看了大佬不少文章，感受还是很明显的

为了加深理解和印象，同时也是为了培养写东西的能力，这一年下来也陆续写了一些文章，算是有了一些思考和收获吧。

# 定2021

我对自己整个2021的目标，一句话概括就是：提升个人技术至以达到冲击大厂的能力😀。

几个方向：

  1. 基础知识进一步巩固扎实
  2. 前端构建工具及工程化方面的学习和研究，深入原理
  3. 主流框架及库的熟练使用，并尝试源码阅读和分析，理解运行逻辑
  4. 融入更多的前端圈子，包括沿技术资讯相关，形一套成自己的知识获取途径
  5. 服务器及后端相关知识学习：Docker、Java、Go、Python、Node等

一开始确实是啥都想学，啥都想做的心理，但是很难在一件事情上按照规划坚持执行下去，经常是这个做一点，碰到另一个后，也想做一点，没有做到完全的投入，或者说没有很好的学习规划导致，这也是需要提高和加强的点。

比较理想的是学习与趣味共同相辅相成，如果只是一味的搞业务、读源码，说实话，我顶不住，也坚持不了。

劳逸结合才是正途，总不能一天到晚都在学习吧，能这样想的，几乎都很难做到，至少我是不行😂，打打游戏，出去玩玩之类的（我比较喜欢看电影），或者搞搞自己的小项目，比较有趣的想法实践。

接下来对于学习和个人时间的管理上是需要仔细研究一下。

# 收获

  • 前端工程化和规范是做了一些，有点心得体会，算是小有收获吧，产出了几个实战项目
  • 用NodeJS写了一些小工具，但也就局限于一些常用API的使用上，熟练使用还需加油
  • 源码方面目前看的都是偏小的一些库，以及成熟方案思路的借鉴
  • 围绕公众号、GitHub、知名大牛、周刊订阅建设了一个还算可以的前端圈子以及快速获取信息的能力
  • TypeScript日常使用能力达成，并习惯为第一开发语言
  • vite、vue3使用并实战
  • 解决问题的能力，和思考问题的方式有一定的提高
  • GitHub使用的正确姿势正在完善，参与开源
  • 个人博客原创文章20篇，公众号也在运营并同步更新文章
  • 2021年开始写掘金，目前累计输出11篇文章

就收获而言，也不太好量化，仔细回想起来也没觉得做了什么，看上去也比较水，感觉挺虚的，看得出来还是有不少差距的。

要说感受比较明显的一点，应该就是思考问题的能力和方式了，我个人觉得对于解决问题的能力是有所提升的，一方面来源于经验的增长，一方面来源于技术的积累，而另一方则来自于对于源码的学习，在遇到问题时可以从底层思维的角度去思考，这对解决问题是有着极大的帮助。

现阶段就是不断地积累，等到我们储备的量够了的时候，便可融会贯通（数量改变质量）

# 不足

首先，是目标没达成的部分，或不足的

  • 前端工程化，只在使用方面，但是没有深入原理，也就只是用法方面熟悉些，接下来需要深入原理
  • 源码学习还有待提高，很多都是浅尝辄止，这需要比较投入的持续学习
  • TypeScript虽说去年一整年几乎都在用，但是现在想想，大多停留在基础用法上面，最佳实践需要再提升
  • 文章、博客之前调整过一次，目前只保留了原创，不过对于内容质量和深度还是有待提高，现阶段还是学习积累为主，文章以辅，主要作为记录和写作训练使用

自身实力明显跟不上自身期望啊，还需加油！

# 新年

首先，2022新年快乐~

新的一年新的起点，重点是备战向大厂冲击，围绕该方向以及自身规划确定以下主要方向：

  • 基础
  • 源码
  • 算法
  • 体系

目标规划：

  • 基础知识还需加强，这应是个长期持续的工作
  • 需要形成自己的知识体系和框架
  • webpack原理及手写、babel原理及手写
  • 红宝书、小黄书、设计模式、算法小抄啃完
  • 算法训练、LeetCode刷起来

次要目标：

  • TypeScript需要达到熟练使用，并进一步补强
  • 服务器相关知识，就目前而言优先级不高，需要会用
  • 博客使用astrojs、nextjs、nuxtjs重构
  • nestjs入门、koa等后端框架使用以及MySQL入门使用
  • Python、GO语言入门，Docker基本使用
  • 按规划完成目标，在时间允许的情况下完成规划的个人项目
